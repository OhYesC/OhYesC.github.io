概述



右半边红色部分

  • WebSocketHandlerMapping 是 spring-websocket 项目中的类

AbstractUrlHandlerMapping

实现 MatchableHandlerMapping 接口，继承 AbstractHandlerMapping 抽象类，以 URL 作为 Handler 的 HandlerMapping 抽象类，提供 Handler 的获取、注册等等通用的骨架方法。

构造方法

registerHandler

注册多个 URL 的处理器

registerHandler

  • 如果非延迟加载，并且 handler 为 String 类型，并且还是单例，则去获取 String 对应的 Bean 对象。
  • 如果已经存在，并且和 resolvedHandler 不同，则抛出 IllegalStateException 异常。
  • 如果是 / 根路径，则设置处理器为 rootHandler
  • 如果是 /* 路径，则设置处理器为默认处理器
  • 添加到 handlerMap 中

getHandlerInternal

获得处理器

  • buildPathExposingHandler(handler, bestMatch, pathWithinMapping, uriTemplateVariables) 方法，构建暴露路径的 Handler

lookupHandler

获得处理器

  • 直接匹配

    • 从 handlerMap 中，直接匹配处理器
    • 如果找到的处理器是 String 类型，则从容器中找到 String 对应的 Bean 类型作为处理器
    • 校验处理器
    • 调用 #buildPathExposingHandler(handler, bestMatch, pathWithinMapping, uriTemplateVariables) 方法，构建暴露路径的 Handler

  • 模式匹配

    • 遍历 handlerMap 集合，逐个 Pattern 匹配合适的，并添加到 matchingPatterns 中
    • 获得首个匹配的结果 patternComparator
    • 获得 bestMatch 对应的处理器
    • 如果找到的处理器是 String 类型，则从容器中找到 String 对应的 Bean 类型作为处理器。
    • 调用 #validateHandler(Object handler, HttpServletRequest request) 方法，空方法，校验处理器。
    • 获得匹配的路径
    • 获得路径参数集合uriTemplateVariables。因为，可能存在多个最佳匹配，所以每个都会比较一次，全部添加到 uriTemplateVariables 中。

  • 都不匹配

    • 都不匹配，返回 null

buildPathExposingHandler

  • 创建了 HandlerExecutionChain 对象
  • 添加 PathExposingHandlerInterceptor 和 UriTemplateVariablesHandlerInterceptor 拦截器，用途在于通过这两个拦截器，暴露 bestMatchingPattern 和 uriTemplateVariables 到请求的属性中。

PathExposingHandlerInterceptor ，继承 HandlerInterceptorAdapter 类，是 AbstractUrlHandlerMapping 的内部类

UriTemplateVariablesHandlerInterceptor ，继承 HandlerInterceptorAdapter 类，是 AbstractUrlHandlerMapping 的内部类

match

initApplicationContext

初始化

自动探测处理器

  • 获得 Bean 的名字的数组
  • 遍历 Bean ，逐个注册

调用 #determineUrlsForHandler(String beanName) 抽象方法，获得 Bean 对应的 URL 们

  • 具体的探索逻辑，还是交给子类处理。
  • 如果 URL 们非空，则调用父类 AbstractUrlHandlerMapping 的 #registerHandler(String[] urlPaths, String beanName) 方法，执行注册处理器。
